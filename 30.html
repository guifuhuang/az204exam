Azure Service Bus 与 Event Grid 集成配置

您需要配置 Azure Service Bus 和 Azure Event Grid 的集成。

应该如何完成 CLI 语句？请从答案区域选择适当的选项。

注意：每个正确选择得 1 分。
az
create --source-resource-id $topicid --name $name --endpoint-type
--endpoint $endpoint
✅ 正确答案与解析
az eventgrid event-subscription create --source-resource-id $topicid --name $name --endpoint-type servicebusqueue --endpoint $endpoint

解析说明：

根据 Azure 文档中关于 Service Bus 和 Event Grid 集成的内容：

下拉框1：eventgrid - 因为我们要创建的是 Event Grid 的事件订阅，而不是直接操作 Service Bus。

下拉框2：event-subscription - 这是创建事件订阅的正确子命令，用于定义事件源和事件处理程序之间的关联。

下拉框3：servicebusqueue - 当目标端点是 Service Bus 队列时，必须指定 endpoint-type 为 servicebusqueue。这是 Azure 文档中明确要求的配置。

该命令的作用是创建一个 Event Grid 订阅，将来自指定资源 ($topicid) 的事件路由到 Service Bus 队列 ($endpoint)。这种集成模式常用于解耦系统组件，实现可靠的事件传递。
