QUESTION NO: 166 HOTSPOT

A company is developing a Java web app. The web app code is hosted in a GitHub repository located at https://github.com/Contoso/webapp.

The web app must be evaluated before it is moved to production. You must deploy the initial code release to a deployment slot named staging.

You need to create the web app and deploy the code.

How should you complete the commands? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

# Create resource group
create --location centralus --name AppResourceGroup
# Create app service plan
create --name ASP-AppResourceGroup --resource-group AppResourceGroup --sku S1
# Create web app
create --name businesswebapp --resource-group AppResourceGroup --plan ASP-AppResourceGroup --runtime "JAVA:11-java11"
# Create deployment slot
create --name businesswebapp --resource-group AppResourceGroup --slot staging
# Configure deployment source
config --name businesswebapp --resource-group AppResourceGroup --slot staging --repo-url https://github.com/Contoso/webapp --branch master --manual-integration

正确答案:

  1. Create resource group: group
  2. Create app service plan: appservice plan
  3. Create web app: webapp
  4. Create deployment slot: webapp deployment slot
  5. Configure deployment source: webapp deployment source

解析:

根据题目要求和Azure CLI命令规范：

  1. 创建资源组：
    • 使用命令：group create
    • 这是Azure CLI中创建资源组的命令前缀
    • 完整命令：az group create
  2. 创建应用服务计划：
    • 使用命令：appservice plan create
    • 这是创建应用服务计划的命令前缀
    • 完整命令：az appservice plan create
  3. 创建Web应用：
    • 使用命令：webapp create
    • 这是创建Web应用的命令前缀
    • 完整命令：az webapp create
    • 指定Java 11运行时环境
  4. 创建部署槽：
    • 使用命令：webapp deployment slot create
    • 这是创建部署槽的命令前缀
    • 完整命令：az webapp deployment slot create
  5. 配置部署源：
    • 使用命令：webapp deployment source config
    • 这是配置部署源的命令前缀
    • 完整命令：az webapp deployment source config
    • 连接到GitHub仓库并指定master分支

完整工作流程说明：

  1. 首先创建资源组作为资源容器
  2. 然后创建应用服务计划定义计算资源
  3. 接着创建Web应用并指定Java运行时
  4. 创建staging部署槽用于预生产测试
  5. 最后配置从GitHub仓库的代码部署
