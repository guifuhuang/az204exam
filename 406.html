您正在使用 Azure Event Grid 实现一个应用程序，以向客户推送近实时信息。

要求：

  • 您必须向数千名客户发送包含数百种事件类型的事件。
  • 事件在处理前必须按事件类型进行过滤。
  • 身份验证和授权必须使用 Microsoft Entra ID 处理。
  • 事件必须发布到单个端点。

您需要实施 Azure Event Grid。

解决方案：

将事件发布到事件域（event domain）。为每个客户创建一个自定义主题（custom topic）。

该解决方案是否满足目标？

✓ 正确答案：A. Yes

解释：

该解决方案满足所有要求：

1
事件域（Event Domain）允许管理大量自定义主题（每个客户一个主题），满足向数千客户发送事件的需求。
2
每个自定义主题可以配置事件过滤规则，支持按事件类型过滤。
3
Azure Event Grid 支持使用 Microsoft Entra ID 进行身份验证和授权。
4
事件域提供单一端点（domain endpoint）来发布所有事件，满足"发布到单个端点"的要求。
