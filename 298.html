QUESTION NO: 298 HOTSPOT

You are developing an ASP.NET Core app that includes feature flags which are managed by Azure App Configuration. You create an Azure App Configuration store named AppFeatureFlagStore as shown in the exhibit:

Configuration Store:

Key Label State Description Last modified
Export Export On Ability to export data. 6/11/2020, 9:13:26 AM

You must be able to use the feature in the app by using the following markup:

HTML Markup:

<feature name="export"> <li class="export-item"> <a class="nav-link text-dark" asp-area="" asp-controller="Home" asp-action="Export" href="/export/Index"> (Export data) </a> </li> </feature>

You went to update the app to use the feature flag. Which values should you use? To answer, select the appropriate options in the answer area. NOTE: Each correct selection is worth one point.

Feature Flag Configuration:

Controller attribute
Startup method
AppConfig endpoint setting

正确答案：

配置项 正确选项 说明
Controller attribute FeatureGate FeatureGate是专门用于功能开关控制的特性
Startup method AddAzureAppConfiguration 需要添加Azure应用配置服务以使用功能标志
AppConfig endpoint setting https://appfeatureflagstore.azconfig.io 应使用配置存储的实际端点(azconfig.io域名)

技术说明：

1. Controller attribute：选择"FeatureGate"因为这是ASP.NET Core中专门用于功能开关控制的特性，它会检查Azure App Configuration中的功能标志状态。

2. Startup method：选择"AddAzureAppConfiguration"因为这是将Azure应用配置服务集成到ASP.NET Core应用的标准方法。

3. AppConfig endpoint setting：选择"https://appfeatureflagstore.azconfig.io"因为这是Azure App Configuration的标准端点格式，且与题目中提到的配置存储名称一致。

这些配置组合确保：

  • 应用能够正确连接到Azure App Configuration
  • 功能标志状态能够被正确读取和应用
  • HTML中的feature标签能够正常工作
  • Export功能可以根据配置动态开启或关闭
