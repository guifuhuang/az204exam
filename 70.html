Azure 网站配置测试题

Topic 8: Munson's Pickles and Preserves Farm

问题 70: 配置企业网站的所有站点设置

场景描述

Munson's Pickles and Preserves Farm 正在将企业网站迁移到 Azure App Service。迁移需要满足特定的安全要求，包括集中存储和保护所有站点设置。

配置要求

  • 集中存储和保护所有站点设置，不使用密钥
  • 静态和传输中的设置必须加密
  • 使用 Azure 资源的托管标识进行 Microsoft Entra 身份验证
  • 应用最小权限原则

当前挑战

开发人员在本地调试期间报告了 Service Bus 身份验证错误，需要确保配置的安全性和集中管理。

请选择三个正确的操作（多选）

A

创建托管标识

B

更新 Azure App Configuration 存储的角色分配

C

创建 Azure Key Vault

D

创建 Azure App Configuration 存储

E

更新 Azure Key Vault 的角色分配

正确答案与解析

正确答案: A, D, B

创建托管标识，创建 Azure App Configuration 存储，更新 Azure App Configuration 存储的角色分配

解析:

此解决方案满足所有配置要求：

  • 创建托管标识 (A) 实现无密钥访问，满足"不使用密钥"的要求
  • 创建 Azure App Configuration 存储 (D) 提供集中存储设置的能力
  • 更新 Azure App Configuration 存储的角色分配 (B) 应用最小权限原则，确保安全访问

为什么选择此方案：

  • Azure App Configuration 专门设计用于集中管理应用程序设置
  • 托管标识提供安全的无密钥身份验证，符合"使用托管标识"的要求
  • 角色分配确保只有授权资源可以访问配置
  • App Configuration 自动加密静态和传输中的数据

为什么其他选项不正确：

  • C 和 E (Key Vault 相关)：Key Vault 用于存储密钥和密码，而文档要求"不使用密钥"存储站点设置
  • Key Vault 更适合存储敏感凭证，而不是常规应用配置
  • 文档明确要求"集中存储和保护所有站点设置"，App Configuration 是更合适的服务

此方案的优势：

  • 完全符合"集中存储和保护所有站点设置，不使用密钥"的核心要求
  • 解决开发人员本地调试期间的身份验证错误
  • 支持应用最小权限原则
  • 简化配置管理和部署流程
