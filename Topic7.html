Background

VanArsdel Ltd. is a global office supply company. The company is based in Canada and has retail store locations across the world. The company is developing several cloud-based solutions to support their stores, distributors, suppliers, and delivery services.

Current Environment

Corporate Website

The company provides a public website located at http://www.vanaisdelttd.com. The website consists of a React JavaScript user interface, HTML, CSS, image assets, and several APIs hosted in Azure functions.

Requirements

Corporate Website Requirements

  • Secure the website by using SSL
  • Minimize costs for data storage and hosting
  • Implement native GitHub workflows for continuous integration and continuous deployment (CI/CD)
  • Distribute the website content globally for local use
  • Implement monitoring using Application Insights and availability web tests including SSL certificate validity and custom header value verification
  • The website must have 99.95 percent uptime

Retail Store Locations Requirements

  • Azure Functions must process data immediately when data is uploaded to Blob storage
  • Azure Functions must update Azure Cosmos DB using native SQL language queries
  • Audit store sale transaction information nightly to validate data, process sales financials, and reconcile inventory
  • Perform point-in-time restoration of retail store location data due to unexpected data deletion
  • Address high Request Unit (RU) usage and point read latency issues in Azure Cosmos DB queries

Delivery Services Requirements

  • Store service telemetry data in Azure Cosmos DB using an Azure Function
  • Data must include: item ID, delivery vehicle license plate, vehicle package capacity, and current vehicle location coordinates
  • Store delivery driver profile information in Azure Active Directory (Azure AD) using an Azure Function called from the corporate website

Inventory Services Requirements

  • Provide third-party API access to a specific blob within retail store storage account for three months
  • Access must be read-only

Security Requirements

  • Centralize management and distribution of configuration data for all Azure Functions across environments and geographies
  • Encrypt configuration data using company-provided RSA-HSM key
  • Use Azure AD for authentication and authorization
  • Use managed identities for services where possible
