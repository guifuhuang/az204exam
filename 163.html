Azure Batch 简介

Azure Batch 是微软提供的云规模作业调度和计算管理服务，主要用于运行大规模并行和高性能计算 (HPC) 应用程序。主要特点包括：

  • 大规模并行计算：可以同时管理数千个计算节点
  • 自动缩放：根据工作负载自动调整计算资源
  • 灵活的作业调度：支持复杂的任务依赖关系和执行计划
  • 多种应用程序支持：可以运行容器、MPI应用程序或自定义应用程序
  • 成本优化：支持低优先级VM和Spot实例以降低成本

典型应用场景包括：金融风险建模、媒体渲染、科学计算、机器学习训练等需要大量计算资源的场景。

QUESTION NO: 163

You are developing a software solution for an autonomous transportation system. The solution uses large data sets and Azure Batch processing to simulate navigation sets for entire fleets of vehicles.

You need to create compute nodes for the solution on Azure Batch.

What should you do?

正确答案: B

说明:

要在Azure Batch中创建计算节点，正确的做法是：

  • 选项B是正确的，因为BatchClient.PoolOperations.CreatePool方法专门用于创建计算节点池（包含多个计算节点）。
  • 选项A不正确，因为创建Batch账户只是设置管理环境，不会直接创建计算节点。
  • 选项C和D不正确，因为JobAddParameter和TaskAddParameter类用于管理工作和任务，而不是创建计算节点。

在Azure Batch架构中：

  1. Batch账户是管理资源的顶层容器
  2. 计算节点池(Pool)是实际的计算资源集合
  3. 作业(Job)是在池上运行的工作单元
  4. 任务(Task)是作业中的具体计算单元

题目中提到的自动驾驶系统需要大量计算资源进行导航模拟，因此创建计算节点池是最合适的解决方案。
