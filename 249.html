QUESTION NO: 249 HOTSPOT

You have a single page application (SPA) web application that manages information based on data returned by Microsoft Graph from another company's Azure Active Directory (Azure AD) instance.

Users must be able to authenticate and access Microsoft Graph by using their own company's Azure AD instance.

You need to configure the application manifest for the app registration.

How should you complete the manifest? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

{
"oauth2AllowImplicitFlow": ,
"": [{
"resourceAppId": "00000000-0000-0000-0000-000000000000",
"resourceAccess": [{
"id": "24a6cdd6-fabl-4aaf-0168-3cc8225e90d0",
"type": "Scope",
}]}]
"signInAudience": ""
}

Correct Answer and Explanation

Correct Selections:

  1. oauth2AllowImplicitFlow: true
  2. requiredResourceAccess
  3. signInAudience: AzureADMyOrg

中文说明：

1. oauth2AllowImplicitFlow: true：

  • 启用隐式授权流
  • 允许SPA应用直接从授权端点获取令牌
  • 这是单页应用(SPA)的常见认证模式
  • 参考文档：Microsoft隐式授权流文档

2. requiredResourceAccess：

  • 指定应用需要访问的资源(Microsoft Graph)
  • 包含resourceAppId和所需的权限范围(Scope)
  • resourceAppId "00000000-0000-0000-0000-000000000000"代表Microsoft Graph

3. signInAudience: AzureADMyOrg：

  • 只允许本组织的Azure AD用户登录
  • 单租户场景
  • 虽然题目提到"其他公司"，但实际配置应为AzureADMyOrg，因为应用是注册在"另一个公司"的Azure AD中
  • 参考文档：Microsoft应用清单参考

技术说明：

  • 隐式流适用于SPA应用，因为它们在浏览器中运行且无法安全存储客户端密钥
  • requiredResourceAccess配置确保应用有权访问Microsoft Graph API
  • AzureADMyOrg设置限制登录用户必须来自应用注册所在的Azure AD租户
  • resourceAccess中的type为Scope表示委托权限

完整配置示例：

{
  "oauth2AllowImplicitFlow": true,
  "requiredResourceAccess": [
    {
      "resourceAppId": "00000003-0000-0000-c000-000000000000",
      "resourceAccess": [
        {
          "id": "e1fe6dd8-ba31-4d61-89e7-88639da4683d",
          "type": "Scope"
        }
      ]
    }
  ],
  "signInAudience": "AzureADMyOrg"
}
