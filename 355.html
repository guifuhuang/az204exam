Azure Web Apps 自动缩放配置测试题

QUESTION NO: 355

您为一家商业潜水公司开发 Azure Web Apps。法规要求所有潜水员在每次潜水工作开始后每15天填写一次健康问卷。

您需要配置 Azure Web Apps，以便在潜水员填写问卷时扩展实例数量，并在填写完成后缩减实例数量。

您需要配置自动缩放。哪两种自动缩放配置可以实现此目标？（每个正确答案都提供了一个完整的解决方案）

注意：每个正确选项值一分。

答案与解析

正确答案：A 和 D

解析说明：

根据题目要求，需要每15天在特定时间段（潜水员填写问卷期间）扩展实例，填写完成后缩减实例。以下是详细分析：

  • A. Predictive autoscaling（预测性自动缩放） - 这是正确的解决方案。预测性自动缩放使用机器学习分析历史负载模式，预测未来的负载需求。对于每15天发生的周期性事件（如问卷填写），它可以提前预测负载增加并自动扩展实例。
  • D. Fixed date profile（固定日期配置文件） - 这是正确的解决方案。固定日期配置文件允许您设置特定日期和时间段的缩放规则。您可以预先配置每15天的具体日期，在问卷填写期间扩展实例，填写完成后缩减实例。
  • B. CPU usage-based autoscaling（基于CPU使用率的自动缩放） - 这不适用。题目要求基于特定时间表（每15天）而不是实时负载指标进行缩放。基于CPU的缩放无法保证在问卷填写期间准确扩展。
  • C. Recurrence profile（重复周期配置文件） - 这不适用。重复周期配置文件适用于每日、每周或每月等固定周期，但不支持每15天的非标准周期。Azure的重复周期配置无法直接设置为每15天一次。

Azure自动缩放最佳实践：

对于这种周期性可预测的工作负载：

  1. 优先使用预测性自动缩放，它可以根据历史数据自动识别模式
  2. 当有确切日期时，使用固定日期配置文件进行精确控制
  3. 避免使用基于指标的缩放（如CPU），除非负载模式不可预测
  4. 结合使用预测性缩放和固定日期配置可获得最佳效果
