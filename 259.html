Scenario:

You are developing a website that will run as an Azure Web App. Users will authenticate by using their Azure Active Directory (Azure AD) credentials.

You plan to assign users one of the following permission levels for the website: admin, normal, and reader. A user's Azure AD group membership must be used to determine the permission level.

You need to configure authorization.

Solution:

  • Configure and use Integrated Windows Authentication in the website.
  • In the website, query Microsoft Graph API to load the group to which the user is a member.

Does the solution meet the goal?

答案与说明：

正确答案：B. No

说明：

这个解决方案不符合要求，原因如下：

  1. Integrated Windows Authentication (IWA) 主要用于本地Active Directory环境中的Windows身份验证，而不是Azure AD认证的最佳选择。
  2. Azure Web Apps通常使用基于标准的协议（如OpenID Connect）与Azure AD集成，而不是IWA。
  3. 虽然查询Microsoft Graph API可以获取用户组信息，但这不是最有效的方式。更好的方法是使用Azure AD提供的声明(claims)，其中已经包含了用户的组成员信息。
  4. 正确的做法应该是配置Azure AD应用注册，启用组声明，然后在网站中直接使用这些声明来确定用户权限级别。
