Scenario:

You are developing an Azure solution to collect point-of-sale (POS) device data from 2,000 stores located throughout the world. A single device can produce 2 megabytes (MB) of data every 24 hours. Each store location has one to five devices that send data.

You must store the device data in Azure Blob storage. Device data must be correlated based on a device identifier. Additional stores are expected to open in the future.

You need to implement a solution to receive the device data.

Solution: Provision an Azure Event Grid. Configure the machine identifier as the partition key and enable capture.

Question: Does the solution meet the goal?

正确答案: B. No

解释说明:

这个解决方案不符合要求，原因如下：

1. Event Grid 不适合大规模设备数据收集场景：Event Grid 主要用于事件路由和分发，而不是持续的数据流收集[6](@ref)[9](@ref)。它更适合处理离散事件（如设备连接/断开事件），而不是持续的设备遥测数据流[2](@ref)[6](@ref)。

2. 分区键配置不正确：Event Grid 不支持以设备标识符作为分区键的概念[6](@ref)。题目中提到的"配置机器标识符为分区键"在 Event Grid 中无法实现。

3. 数据量考虑：题目描述的场景涉及每天 2,000 个门店（每个门店 1-5 个设备）产生的数据，总数据量可能达到每天 4-20GB。Event Grid 不是为处理这种持续、大量的数据流而设计的[1](@ref)[6](@ref)。

4. 存储要求：题目明确要求将数据存储在 Azure Blob 存储中，而 Event Grid 的捕获功能主要用于将事件存档到 Blob 存储，这不是一个高效的数据收集方案[6](@ref)[9](@ref)。

更合适的解决方案应该是：

  • 使用 Azure IoT Hub 来收集设备数据，然后通过消息路由将数据发送到 Blob 存储[6](@ref)[8](@ref)
  • 或者使用 Azure Event Hubs 作为高吞吐量的数据入口，然后将数据路由到 Blob 存储[6](@ref)[9](@ref)
