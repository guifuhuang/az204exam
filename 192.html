QUESTION NO: 192 HOTSPOT

You are building a website to access project data related to terms within your organization. The website does not allow anonymous access. Authentication performed using an Azure Active Directory (Azure AD) app named internal.

The website has the following authentication requirements:

  • Azure AD users must be able to login to the website.
  • Personalization of the website must be based on membership in Active Directory groups.

You need to configure the application's manifest to meet the authentication requirements.

How should you configure the manifest? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

{
    "appId": "d61126e3-089b-4adb-b721-d5023213df7d",
    "": "All",
    "": true
}

正确答案:

第一个下拉框选择: groupMembershipClaims

第二个下拉框选择: oauth2AllowImplicitFlow

说明:

1. groupMembershipClaims: 这个设置需要配置为"All"以启用组声明，这样应用程序就可以接收用户所属的所有安全组。这是满足"基于Active Directory组成员资格进行网站个性化"要求的关键配置。

2. oauth2AllowImplicitFlow: 需要设置为true以允许隐式授权流程，这对于网站认证是必要的。虽然现代应用推荐使用授权码流程，但隐式流程仍被广泛用于单页应用(SPA)等场景。

其他选项不正确的原因：

  • optionalClaims: 这不是用于组声明的正确属性
  • allowPublicClient: 适用于移动应用等公共客户端，与题目中的网站场景不完全匹配
  • oauth2Permissions/requiredResourceAccess: 这些与组声明和认证流程设置无关
