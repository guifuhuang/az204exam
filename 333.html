QUESTION NO: 333 HOTSPOT

You are developing a web application that uses the Microsoft identity platform for user and resource authentication. The web application calls several REST APIs.

You are implementing various authentication and authorization flows for the web application.

You need to validate the claims in the authentication token.

Which token type should you use? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

Requirement Token type
Identify users for the application by using a JWT token that contains claims.
Identify the permissions granted to APIs by using a JWT token that contains claims.
Provide the web application with long-term access to resources on behalf of users without requiring interaction with those users.
Provide XML representations of claims that can be consumed by applications that use WS-Federation.

正确答案:

Identify users with JWT token: ID

Identify API permissions: Access

Long-term access without user interaction: Refresh

XML claims for WS-Federation: SAML

解释说明:

1. ID令牌 用于识别应用程序的用户。ID令牌是JWT格式，包含用户身份信息，专门用于客户端验证用户身份[2](@ref)[4](@ref)。

2. Access令牌 用于识别API权限。Access令牌包含授予API的权限范围，用于授权访问受保护的资源[1](@ref)[7](@ref)。

3. Refresh令牌 提供长期访问资源的能力而无需用户交互。Refresh令牌用于获取新的Access令牌，有效期较长[3](@ref)[9](@ref)。

4. SAML令牌 是XML格式的安全令牌，支持WS-Federation协议。它能够提供XML格式的声明表示，被广泛用于企业身份联合场景[10](@ref)。

这些选择是基于以下原因：

  • ID令牌专门设计用于传递用户身份信息[2](@ref)
  • Access令牌包含API权限范围信息[7](@ref)
  • Refresh令牌提供长期访问能力[9](@ref)
  • SAML是WS-Federation协议的标准XML令牌格式[10](@ref)
