QUESTION NO: 412

You develop an Azure App Service web app and deploy to a production environment. You enable Application Insights for the web app.

The web app is throwing multiple exceptions in the environment.

You need to examine the state of the source code and variables when the exceptions are thrown.

Which Application Insights feature should you configure?

正确答案:

B. Snapshot Debugger

说明:

为什么选择Snapshot Debugger:

  1. Snapshot Debugger是专门用于捕获生产环境中异常发生时应用程序状态的工具
  2. 它会在异常发生时自动收集：
    • 调用堆栈信息
    • 局部变量的值
    • 源代码执行位置
  3. 这些快照可以在不停止应用的情况下进行调试分析
  4. 完美匹配题目要求的"检查异常发生时源代码和变量的状态"

功能对比:

功能 用途 是否满足需求
Smart detection 自动检测性能异常和故障模式 ❌ 仅能发现问题，不能查看代码状态
Snapshot Debugger 捕获异常时的应用程序状态 ✅ 完美匹配需求
Standard test 基本可用性测试 ❌ 不提供调试信息
Profiler 性能瓶颈分析 ❌ 用于性能优化而非异常调试

配置Snapshot Debugger的步骤:

  1. 在Application Insights资源中启用Snapshot Debugger
  2. 在应用服务配置中添加连接字符串
  3. 部署应用程序时包含调试符号(PDB文件)
  4. 设置快照收集规则(如异常类型、频率限制等)

关键优势:

  • 无需在生产环境中直接调试，降低风险
  • 捕获真实用户场景中的异常状态
  • 支持.NET和Java应用程序
  • 与Visual Studio集成，可直接在IDE中查看快照
