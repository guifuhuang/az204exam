QUESTION NO: 346 HOTSPOT

You are developing an Azure Durable Function to manage an online ordering process.

The process must call an external API to gather product discount information.

You need to implement Azure Durable Function.

Which Azure Durable Function types should you use? Each correct answer presents part of the solution.

NOTE: Each correct selection is worth one point.

正确答案:

A. Orchestrator

B. Entity

解释说明:

1. Orchestrator (业务流程协调程序函数) 是管理在线订购流程的核心组件。它负责定义操作的执行顺序和方式，可以协调多个活动函数和外部API调用[3](@ref)[5](@ref)。在这个场景中，Orchestrator函数将负责调用外部API获取产品折扣信息，并管理整个订单处理流程。

2. Entity (实体函数) 用于管理订单流程中的状态信息。实体函数允许你将状态显式地存储为对象，非常适合管理订单状态、折扣信息等需要持久化的数据[6](@ref)[7](@ref)。在这个场景中，Entity函数可以用来存储和管理从外部API获取的折扣信息。

其他选项不正确的原因：

  • C选项Activity函数虽然可以用于执行具体任务（如调用外部API），但它不负责管理整个流程
  • D选项Client函数主要用于触发业务流程，而不是管理业务流程

这两种函数类型(A和B)组合使用可以很好地满足题目中"管理在线订购流程并调用外部API获取折扣信息"的需求[3](@ref)[6](@ref)。
