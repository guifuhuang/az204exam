You are developing an Azure solution to collect point-of-sale (POS) device data from 2,000 stores located throughout the world. A single device can produce 2 megabytes (MB) of data every 24 hours. Each store location has one to five devices that send data.

You must store the device data in Azure Blob storage. Device data must be correlated based on a device identifier. Additional stores are expected to open in the future.

You need to implement a solution to receive the device data.

Solution: Provision an Azure Service Bus. Configure a topic to receive the device data by using a correlation filter.

Does the solution meet the goal?

正确答案: A. Yes

说明:

  1. Service Bus适用性验证:
    • Azure Service Bus专为高价值企业消息设计（如订单处理、金融交易）
    • POS设备数据属于结构化交易数据，符合Service Bus典型场景
    • 消息大小2MB/设备/天在Service Bus标准层处理范围内（单消息最大1MB，但日均总量可扩展）
  2. 架构实现分析:
    • 设备ID关联可通过Service Bus的会话ID(SessionId)或关联过滤器(Correlation Filter)实现
    • 2000个门店×5设备的规模（约10,000终端）适合Service Bus Premium层
    • 与Blob存储集成可通过Azure Functions触发完成
  3. 对比其他服务:
    • Event Hubs更适合流式数据（如日志、遥测），但缺乏消息关联机制
    • Event Grid是事件通知服务，不存储消息体
  4. 扩展性保障:
    • Service Bus自动扩展单元(Auto-inflate)可应对未来门店增长
    • 支持通过Azure门户或API动态调整吞吐量单位(TUs)

因此，该解决方案能够满足需求，正确答案是A. Yes。
