Azure 数据库连接字符串检索测试题

QUESTION NO: 25 HOTSPOT

您需要检索数据库连接字符串。应该使用哪些值？

根据以下要求选择适当的选项：

要求：

  • 数据库连接字符串存储在 Azure Key Vault 中
  • Key Vault 名称: cpandlkeyvault
  • 密钥名称: PostgreSQLConn
  • ID: 80df3e46ffcd4f1cb187f79905e9a1e8
  • 连接信息经常更新，应用程序必须始终使用最新信息连接数据库
配置连接字符串检索参数
RESTAPI Endpoint:
https:// .vault.azure.net/secrets/

Variable type to access Azure Key Vault secret values:

正确答案：

Key Vault 名称: cpandlkeyvault

密钥名称: PostgreSQLConn

变量类型: Environment

说明：

根据 Azure Key Vault 的 REST API 规范：

  • REST API 端点格式应为 https://{vault-name}.vault.azure.net/secrets/{secret-name}
  • 正确的 Key Vault 名称是 cpandlkeyvault，密钥名称是 PostgreSQLConn
  • 虽然选项中未直接提供"cpandlkeyvault"，但根据文档要求，这是正确的Key Vault名称
  • 连接字符串在 Azure 中存储时始终加密（静态加密）

在应用程序中访问连接字符串的正确方式：

  • 对于 PostgreSQL 数据库，连接字符串在运行时可通过环境变量 POSTGRESQLCONNSTR_ 前缀访问
  • 在 .NET 应用中，应使用 ConfigurationManager.ConnectionStrings["PostgreSQLConn"].ConnectionString
  • 由于连接信息经常更新，应直接从 Key Vault 获取最新值，而不是缓存
  • 使用 Environment 变量类型是最安全的方式，因为它不会暴露在客户端代码中

错误选项分析：

  • 第一个选项中的"cpandkewault"是拼写错误，不是正确的Key Vault名称
  • 使用"Session"或"ViewState"存储敏感信息不安全，因为可能暴露给客户端
  • "QueryString"是最不安全的方式，因为连接字符串会直接暴露在URL中

City Power & Light 案例背景

根据文档内容：

  • 企业网站运行在Azure App Service on Linux上
  • 网站使用Azure Storage存储静态内容，并通过Azure CDN分发
  • 解决方案必须满足GDPR合规要求
  • 所有SSL证书和凭证必须存储在Azure Key Vault中
  • 文件访问必须限制访问权限
