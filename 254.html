QUESTION NO: 254 HOTSPOT

You are building a website that is used to review restaurants. The website will use an Azure CDN to improve performance and add functionality to requests.

You build and deploy a mobile app for Apple iPhones. Whenever a user accesses the website from an iPhone, the user must be redirected to the app store.

You need to implement an Azure CDN rule that ensures that iPhone users are redirected to the app store.

How should you complete the Azure Resource Manager template? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

"conditions":[{
"name": "IsDevice",
"parameters": {
"@odata.type": "#Microsoft.Azure.Cdn.Models.",
"operator": "Equal",
"matchValues": [""]
}},
{
"name": "RequestHeader",
"parameters": {
"@odata.type": "#Microsoft.Azure.Cdn.Models.",
"operator": "Contains",
"selector": "",
"matchValues": [""]
}}
]

Correct Answer and Explanation

Correct Selections:

  1. First @odata.type: DeliveryRulesDeviceConditionParameters
  2. First matchValues: iPhone
  3. Second @odata.type: DeliveryRuleRequestHeaderConditionParameters
  4. selector: HTTP_USER_AGENT
  5. Second matchValues: iPhone

中文说明：

1. 第一个@odata.type: DeliveryRulesDeviceConditionParameters

  • 选择设备检测条件参数类型
  • 这是专门用于检测设备类型的参数类型
  • 能准确识别iPhone设备

2. 第一个matchValues: iPhone

  • 精确匹配iPhone设备
  • 使用Equal操作符确保完全匹配
  • 直接识别设备类型，不依赖User-Agent

3. 第二个@odata.type: DeliveryRuleRequestHeaderConditionParameters

  • 选择请求头条件参数类型
  • 用于检查HTTP请求头信息
  • 作为设备检测的补充验证

4. selector: HTTP_USER_AGENT

  • 选择User-Agent请求头
  • User-Agent包含设备信息
  • 可以检测各种浏览器发送的iPhone标识

5. 第二个matchValues: iPhone

  • 匹配包含"iPhone"的User-Agent
  • 使用Contains操作符确保能匹配各种变体
  • 双重验证提高识别准确性

技术说明：

  • 双重验证机制确保准确识别iPhone用户
  • 设备检测(IsDevice)提供直接可靠的识别
  • User-Agent检测(RequestHeader)作为备用验证
  • Equal操作符确保设备类型精确匹配
  • Contains操作符确保能匹配各种User-Agent字符串中的iPhone标识
  • 这种组合方式既可靠又全面

完整规则示例：

"conditions":[{
  "name": "IsDevice",
  "parameters": {
    "@odata.type": "#Microsoft.Azure.Cdn.Models.DeliveryRulesDeviceConditionParameters",
    "operator": "Equal",
    "matchValues": ["iPhone"]
  }},
  {
  "name": "RequestHeader",
  "parameters": {
    "@odata.type": "#Microsoft.Azure.Cdn.Models.DeliveryRuleRequestHeaderConditionParameters",
    "operator": "Contains",
    "selector": "HTTP_USER_AGENT",
    "matchValues": ["iPhone"]
  }}
]
