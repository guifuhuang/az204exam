QUESTION NO: 311

You are developing a mobile app that uses an API which stores geospatial data in Azure Cosmos DB.

The app will be used to:

  • Find restaurants in a particular area
  • Display related information including food types and menu information
  • Calculate the optimal route to a selected restaurant from the user's current location

Which Azure Cosmos DB API should you use for the API?

正确答案：

D. Core (SQL)

技术说明：

1. MongoDB API 最适合地理空间应用的原因：

  • 原生支持地理空间查询运算符（如$near, $geoWithin），可直接查询餐厅与用户位置的距离[5](@ref)
  • 文档模型天然适合存储餐厅信息（菜单、菜系等嵌套JSON结构）[2](@ref)
  • 与移动开发框架（如Node.js、.NET）集成良好，简化API开发[6](@ref)
  • 支持与Azure Maps服务无缝集成实现路线规划[1](@ref)

2. 典型数据模型示例：

{
  "_id": "restaurant123",
  "name": "Azure Bistro",
  "location": {  // GeoJSON格式
    "type": "Point",
    "coordinates": [-122.12, 47.67]
  },
  "cuisine": ["French", "Italian"],
  "menu": [
    {"item": "Salmon", "price": 25},
    {"item": "Pasta", "price": 18}
  ]
}

3. 其他API的限制：

  • Core (SQL)：虽支持地理查询，但文档结构灵活性不如MongoDB[1](@ref)
  • Gremlin：适合社交网络关系分析，地理空间功能有限[4](@ref)
  • Cassandra：缺乏原生地理空间支持，需额外处理坐标数据[2](@ref)

4. 解决方案优势：

  • 开发效率：使用熟悉的MongoDB查询语法快速实现功能[5](@ref)
  • 性能优化：自动索引地理坐标字段，确保毫秒级响应[1](@ref)
  • 全球扩展：利用Cosmos DB多区域复制保障低延迟访问[10](@ref)
