Azure 航运数据迁移方案测试题

QUESTION NO: 6

您需要将本地航运数据迁移到 Azure。
您应该使用什么？
案例背景： 将本地航运应用程序迁移到Azure，航运数据使用MongoDB JSON文档存储数据库，迁移过程需要最小化成本和停机时间。
A
Azure Migrate
B
Azure Cosmos DB 数据迁移工具 (dt.exe)
C
AzCopy
D
Azure 数据库迁移服务

航运数据迁移要求：

  • 数据存储在 MongoDB JSON 文档数据库中
  • 迁移过程需要最小化成本
  • 迁移过程需要最小化停机时间
  • 支持迁移到 Azure Cosmos DB

正确答案与解析

正确答案：D. Azure 数据库迁移服务

解析说明：

根据文档中的参考答案和航运数据迁移的要求，Azure 数据库迁移服务是最合适的解决方案：

1. MongoDB 迁移支持：Azure 数据库迁移服务专门支持从本地或云实现的 MongoDB 迁移到 Azure Cosmos DB，完美匹配文档中描述的 MongoDB JSON 文档存储数据库。

2. 最小化停机时间：该服务支持在线迁移，可以在迁移过程中保持源数据库运行，实现接近零停机时间的迁移，满足"最小化停机时间"的要求。

3. 可靠性和扩展性：Azure 数据库迁移服务能够执行弹性迁移，处理大规模 MongoDB 数据迁移，同时保持高可靠性。

4. 成本效益：作为 Azure 平台服务，它提供成本优化的迁移方案，符合"最小化成本"的要求。

5. 与其他选项的对比：

  • A. Azure Migrate：主要用于评估和迁移虚拟机、应用程序和数据库整体，不适合专门的文档数据库迁移。
  • B. Azure Cosmos DB 数据迁移工具：虽然支持 MongoDB 迁移，但不如 Azure 数据库迁移服务提供完整的在线迁移能力和企业级支持。
  • C. AzCopy：主要用于在 Azure 存储账户之间复制数据，不支持 MongoDB 到 Cosmos DB 的直接迁移。

6. 官方文档支持：Microsoft 官方文档明确指出，使用 Azure 数据库迁移服务可以实现 MongoDB 到 Azure Cosmos DB 的在线和离线迁移。

因此，Azure 数据库迁移服务(D)是满足所有要求的最佳解决方案。
