灾难恢复需求实现 - 代码片段配置
灾难恢复场景要求

• 区域性中断不得影响应用程序可用性

• 所有灾难恢复操作不得依赖于应用程序运行

• 必须确保灾难恢复区域中的数据是最新的

可用的代码片段

true
SingleTransferContext
ShouldTransferCallbackAsync
false
DirectoryTransferContext
ShouldOverwriteCallbackAsync

提示：将代码片段拖到右侧代码块的空白区域

答案区域

PC16var copyOptions = new CopyOptions() { };
PC17var context = new [Value1] { source, destination => Task.FromResult(true) };
PC18context.[Value2] = (source, destination) => Task.FromResult(true);
PC19await TransferManager.CopyAsync(blob, GetBlob(blob), isServiceCopy: [Value3], context: context, options: copyOptions);

正确答案与解析

正确答案：Value1: DirectoryTransferContext | Value2: ShouldTransferCallbackAsync | Value3: false

解析：

根据灾难恢复要求和Azure最佳实践：

正确配置

var context = new DirectoryTransferContext { ... };
context.ShouldTransferCallbackAsync = ...;
isServiceCopy: false

参数说明

Value1: DirectoryTransferContext
Value2: ShouldTransferCallbackAsync
Value3: false

详细解释：

  • DirectoryTransferContext - 用于传输目录中的所有文件（我们传输目录中的所有文件）
  • ShouldTransferCallbackAsync - DirectoryTransferContext.ShouldTransferCallbackAsync委托回调被调用以判断是否应执行传输
  • false - 如果要使用Copy中的重试策略，并希望复制在中断时可以从中间恢复，可以使用SyncCopy（isServiceCopy = false）

注意：如果选择使用服务端复制（'isServiceCopy'设置为true），Azure（目前）不提供SLA。将'isServiceCopy'设置为false将在本地下载源blob，然后将其上传到目标。
