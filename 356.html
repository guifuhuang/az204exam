Azure Functions 配置测试题

QUESTION NO: 356 HOTSPOT

You are developing a C++ application that compiles to a native application named process.exe.

The application accepts images as input and returns images in one of the following image formats: GIF, PNG, or JPEG.

You must deploy the application as an Azure Function.

You need to configure the function and host json files.

How should you complete the json files? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

function.json
{
, "direction": "out", "name": "result" }
host.json
{
"defaultExecutablePath": "process.exe" },
} }
答案与解析

正确答案：

function.json: "type":"http"

host.json: "customHandler": {"description": { 和 "enableForwardingHttpRequest": true

解析：

function.json 配置说明：

1. "type":"http" - 由于应用程序需要处理HTTP请求并返回图像，因此需要配置HTTP绑定类型。

host.json 配置说明：

1. "customHandler": {"description": { - 这是配置自定义处理程序的正确结构，用于指定原生应用程序的处理方式。

2. "enableForwardingHttpRequest": true - 需要设置为true以启用HTTP请求转发功能，这样Azure Functions才能将传入的HTTP请求正确转发给本机应用程序process.exe处理。

整体配置原理：

此配置允许Azure Functions作为托管环境，将传入的HTTP请求（包含图像数据）转发给本机C++应用程序process.exe进行处理。处理完成后，应用程序会返回处理后的图像（GIF、PNG或JPEG格式）。

关键点在于正确配置HTTP绑定类型作为输出，并启用请求转发功能，使Azure Functions能够与本机应用程序正确交互。

配置详解：

  • function.json：定义了函数的绑定信息
    • "type":"http"：指定绑定类型为HTTP
    • "direction":"out"：表示这是输出绑定
    • "name":"result"：绑定参数的名称
  • host.json：配置函数应用级别的设置
    • "customHandler"：启用自定义处理程序
    • "defaultExecutablePath":"process.exe"：指定要执行的原生应用程序
    • "enableForwardingHttpRequest":true：启用HTTP请求转发
