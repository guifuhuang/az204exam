QUESTION NO: 308

You are developing a SaaS application that stores data as key value pairs.

You must make multiple editions of the application available. In the lowest cost edition, the performance must be best-effort, and there is no regional failover.

In higher cost editions customers must be able to select guaranteed performance and support for multiple regions. Azure costs must be minimized.

Which Azure Cosmos DB API should you use for the application?

正确答案：

D. Table API

技术说明：

1. Table API 最适合键值对存储场景的原因：

  • 专门为键值对数据模型设计，与题目要求完全匹配[2](@ref)
  • 基础成本最低，适合低成本版本[7](@ref)
  • 支持分层定价模式，可以根据不同版本需求灵活调整[9](@ref)
  • 与Azure Table Storage兼容，迁移成本低[2](@ref)

2. 多版本支持能力：

  • 低成本版本：可以使用无服务器模式或最低配置的预置吞吐量[7](@ref)
  • 高成本版本：可以启用多区域写入和更高性能级别[6](@ref)
  • 通过同一API支持不同级别的服务，保持代码一致性[2](@ref)

3. 成本优化优势：

  • 存储成本按实际使用量计费，没有最低限制（非空容器除外）[7](@ref)
  • 吞吐量可以精确配置，避免资源浪费[9](@ref)
  • 免费层提供25GB存储和1000 RU/s的免费额度[7](@ref)

4. 其他选项的限制：

  • Core (SQL)：虽然功能全面，但成本较高，不适合简单的键值对场景[1](@ref)
  • MongoDB：更适合文档结构数据，不是最优的键值对解决方案[1](@ref)
  • Cassandra：适合宽列存储场景，对于简单键值对过于复杂[6](@ref)
