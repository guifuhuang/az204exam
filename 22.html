问题 #22 - Azure Functions 托管模型选择

您需要部署名为 CheckUserContent 的 Azure 函数。解决方案必须满足安全性和成本要求。

应该使用哪种托管模型？

安全要求

  • 任何可通过 Internet 访问的 Web 服务必须受到跨站点脚本攻击的保护
  • 所有网站和服务必须使用来自有效根证书颁发机构的 SSL
  • Azure 存储访问密钥必须仅存储在内存中，并且仅对服务可用
  • 所有内部服务必须只能从内部虚拟网络 (VNet) 访问
  • 系统的所有部分都必须支持入站和出站流量限制
  • 所有服务调用必须使用 Azure AD 进行身份验证

成本要求

  • 必须最小化所有 Azure 服务的成本
选择正确的托管模型
A
B
C

正确答案: B. 高级计划 (Premium plan)

高级计划是满足所有安全和成本要求的最佳选择。

详细解析

选择高级计划的原因如下：

  • VNet 集成支持：高级计划原生支持虚拟网络集成，满足"所有内部服务必须只能从内部虚拟网络访问"的安全要求
  • 成本优化：与始终运行的 App Service 计划不同，高级计划支持弹性扩展（包括缩放到零），在空闲时不会产生费用，满足最小化成本的要求
  • 安全合规：高级计划支持私有终结点、Azure AD 身份验证和内存密钥管理，满足所有安全要求
  • 无冷启动：高级计划提供预热的实例，确保内容检查函数在需要时能够快速响应
  • 扩展能力：高级计划支持自动扩展，可以处理用户生成内容的高峰负载

为什么不选择其他选项？

  • 消耗计划 (A)：虽然成本最低，但不支持VNet集成，无法满足内部服务仅限内部VNet访问的安全要求
  • 应用服务计划 (C)：虽然支持VNet集成，但需要持续运行实例（即使没有流量），成本较高，不符合最小化成本的要求
