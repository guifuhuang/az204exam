Azure 解决方案架构师考试题

QUESTION NO: 5

您需要满足航运逻辑应用（Shipping Logic App）的要求。
您应该使用什么？
案例背景： 将本地航运应用程序迁移到Azure，需要支持X12消息格式，保护资源到企业VNet，使用专用存储资源，并维护本地连接。
A
Azure Active Directory 应用程序代理 (Azure Active Directory Application Proxy)
B
点对点 (P2S) VPN 连接 (Point-to-Site (P2S) VPN connection)
C
站点到站点 (S2S) VPN 连接 (Site-to-Site (S2S) VPN connection)
D
本地数据网关 (On-premises Data Gateway)

航运逻辑应用要求：

  • 支持海洋运输和内陆运输工作流
  • 支持行业标准协议 X12 消息格式
  • 将资源保护到企业 VNet，使用具有固定成本模型的专用存储资源
  • 维护本地连接以支持遗留应用程序和最终的 BizTalk 迁移

正确答案与解析

正确答案：D. 本地数据网关 (On-premises Data Gateway)

解析说明：

根据航运逻辑应用的要求，特别是需要维护本地连接以支持遗留应用程序和BizTalk迁移，本地数据网关是最合适的解决方案：

1. 本地连接需求：文档中明确要求"维护本地连接以支持遗留应用程序和最终的BizTalk迁移"。本地数据网关专门用于在Azure服务（如Logic Apps）和本地资源之间建立安全连接。

2. Logic Apps集成：本地数据网关是Azure Logic Apps与本地系统集成的标准解决方案，支持各种协议和数据格式，包括文档中要求的X12消息格式。

3. 安全连接：数据网关提供安全的连接方式，满足"将资源保护到企业VNet"的要求，同时不需要暴露本地网络到公共互联网。

4. 与其他选项的对比：

  • A. Azure AD应用代理：主要用于安全地发布本地Web应用程序，不适合Logic Apps与本地系统的集成。
  • B. P2S VPN：适用于单个客户端连接到VNet，不适合服务器/服务之间的连接。
  • C. S2S VPN：虽然可以建立网络级连接，但不如数据网关轻量级和专门化，且成本更高。

5. 成本考虑：本地数据网关使用固定成本模型，符合"使用专用存储资源与固定成本模型"的要求。

因此，本地数据网关(D)是满足所有要求的最佳解决方案。
