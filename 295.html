QUESTION NO: 295 HOTSPOT

You are developing an ASP.NET Core time sheet application that runs as an Azure Web App.

Users of the application enter their time sheet information on the first day of every month.

The application uses a third-party web service to validate data.

The application encounters periodic server errors due to errors that result from calling a third-party web server. Each request to the third-party server has the same chance of failure.

You need to configure an Azure Monitor alert to detect server errors unrelated to the third-party service. You must minimize false-positive alerts.

How should you complete the Azure Resource Manager template? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

{ "type": "Microsoft.Insights/metricAlerts", "properties": { "criteria": { "odata.type": "...", "allOf": [ { "criterionType": " ", "metricName": " ", "alertSensitivity": " " } ] } } }

正确答案：

配置项 正确选项 说明
criterionType DynamicThresholdCriterion 动态阈值能自动适应流量模式变化，更适合周期性应用
metricName Http5xx 需要监控服务器错误（5xx），而非客户端错误（4xx）
alertSensitivity Low 题目明确要求最小化误报，低敏感度最符合要求

技术说明：

1. criterionType：选择"DynamicThresholdCriterion"因为时间表应用每月第一天使用，流量模式变化大，动态阈值能自动适应这种变化。

2. metricName：选择"Http5xx"因为需要监控服务器端错误（500-599），这些错误通常表示服务器问题而非第三方服务问题。

3. alertSensitivity：选择"Low"因为题目明确要求最小化误报，低敏感度可以减少不必要的警报。

注意：odata.type选项已被移除，因为在新模板中它已被固定为"..."。
