Topic 7: 配送司机身份识别解决方案

案例背景

VanArsdel Ltd.是一家全球办公用品公司，总部位于加拿大，在全球设有零售店。公司正在开发多个基于云的解决方案，以支持其商店、分销商、供应商和配送服务。

配送服务要求：

  • 将服务遥测数据存储在Azure Cosmos DB中
  • 数据包括：物品ID、配送车辆牌照、车辆包裹容量和当前车辆位置坐标
  • 将配送司机档案信息存储在Azure Active Directory (Azure AD)中
  • 需要可靠地识别配送司机档案信息

问题说明： 需要配置系统以可靠地识别配送司机档案信息，确保身份验证的安全性和准确性。

问题 58: 配送司机身份识别配置

您需要可靠地识别配送司机档案信息。应如何配置系统？在答案区域选择适当的选项。

注意：每个正确选择值1分。

JSON Web Token (JWT) Type
Payload Claim Value

您的选择：

JWT 类型: 未选择

Payload Claim: 未选择

正确答案及说明

✓ JWT 类型: ID Token
✓ Payload Claim: oid (Object ID)

说明：

根据案例要求："需要可靠地识别配送司机档案信息"，正确的配置应为：

  • JWT 类型：ID Token - ID令牌包含用户身份信息，专门用于身份验证场景
  • Payload Claim：oid (Object ID) - oid是Azure AD中用户对象的唯一标识符，不会随时间变化，是可靠识别用户的最佳选择

为什么这个配置是最佳的？

  • ID令牌专为身份验证设计，包含用户身份信息
  • Object ID (oid) 是Azure AD中用户对象的唯一标识符
  • oid在用户生命周期内保持不变，即使用户名或电子邮件地址发生变化
  • oid不会暴露个人身份信息，符合安全最佳实践

其他选项分析：

  • Refresh Token：用于获取新的访问令牌，不包含用户身份信息
  • Access Token：用于访问资源，但可能不包含完整的用户身份信息
  • aud (Audience)：标识令牌的目标接收方，不用户
  • idp (Identity Provider)：标识身份提供者，但不直接识别用户

技术要点：

  • ID令牌遵循OpenID Connect标准，专为身份验证设计
  • oid是Azure AD中用户对象的GUID标识符
  • 此配置符合安全要求中的Azure AD认证要求
  • 使用oid可以避免依赖可能变化的用户名或电子邮件地址
