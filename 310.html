QUESTION NO: 310

You are developing a SaaS application that stores data as key-value pairs.

Requirements:

  • Multiple editions of the application must be available
  • In the lowest cost edition:
    • Performance is best-effort
    • No regional failover
  • In higher cost editions:
    • Customers can select guaranteed performance
    • Support for multiple regions
  • Azure costs must be minimized

Which Azure Cosmos DB API should you use for the application?

正确答案：

C. Cassandra

技术说明：

1. Cassandra API 是最佳选择的原因：

  • 原生支持键值对数据模型，同时具备宽列存储的灵活性
  • 低成本版本可使用基本吞吐量配置（最低400 RU/s），实现"best-effort"性能
  • 高成本版本支持：
    • 预配吞吐量（可精确到RU/s级别）
    • 多区域写入（自动同步数据）
    • 性能SLA保障（99.99%可用性）
  • 相比Table API，Cassandra在跨区域场景下的成本优化更好

2. 成本优化实现方案：

  • 低成本版：单区域部署 + 最小RU/s配置（400 RU/s起）
  • 高性能版：多区域自动扩展 + 预留容量折扣
  • 弹性扩展：支持按小时调整吞吐量，应对业务波动

3. 其他API的限制：

  • Core (SQL)：文档模型冗余，基础成本高30%以上
  • MongoDB：适合复杂文档结构，键值场景性价比低
  • Table API：多区域部署成本高于Cassandra

4. Cassandra解决方案优势：

  • 线性扩展：添加节点即可提升性能，适合SaaS多租户场景
  • 成熟生态：与现有Cassandra工具链兼容，降低运维成本
  • 精细控制：可针对不同租户配置独立吞吐量
