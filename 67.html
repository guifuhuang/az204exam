Azure 安全配置测试题

Topic 8: Munson's Pickles and Preserves Farm

问题 67: 保护企业网站以满足安全要求

场景描述

Munson's Pickles and Preserves Farm 正在将企业网站从本地数据中心迁移到 Microsoft Azure。迁移需要满足特定的安全要求，包括保护通信安全、限制内容访问以及应用最小权限原则。

安全要求

  • 使用 TLS/HTTPS 保护所有 Web 通信
  • 根据国家/地区限制 Web 内容以满足合规要求
  • 对所有用户权限和进程访问权限应用最小权限原则
  • 使用 Azure 资源的托管标识进行 Microsoft Entra ID 身份验证

当前问题

企业网站需要安全加固以满足上述要求，同时解决农民报告的 HTTP 503 错误和分销商报告的 HTTP 502 错误。

请选择正确的解决方案

A

创建具有标准计划的 App Service 实例。使用 TLS/SSL 证书配置自定义域。

B

创建带有 Web 应用程序防火墙 (WAF) 的 Azure 应用程序网关。配置端到端 TLS 加密和 WAF。

C

创建 Azure Cache for Redis 实例。更新代码以支持缓存。

D

创建 Azure 内容分发网络配置文件和端点。配置该端点。

正确答案与解析

正确答案: B

创建带有 Web 应用程序防火墙 (WAF) 的 Azure 应用程序网关。配置端到端 TLS 加密和 WAF。

解析:

此解决方案满足所有安全要求：

  • 端到端 TLS 加密 满足使用 TLS/HTTPS 保护所有 Web 通信的要求
  • Web 应用程序防火墙 (WAF) 提供基于国家/地区的内容限制功能，满足合规要求
  • 应用程序网关可以与 Azure 资源的托管标识集成，支持 Microsoft Entra ID 身份验证
  • 此方案同时有助于解决 HTTP 502/503 错误，提供负载均衡和自动扩展能力

其他选项分析：

  • A：仅解决 TLS 要求，但无法满足国家/地区限制内容的要求
  • C：缓存方案主要优化性能，与安全要求无直接关系
  • D：CDN 可提升性能但不提供 WAF 的安全功能，无法满足国家/地区限制要求

文档中明确要求："Secure all web communications using TLS/HTTPS" 和 "Restrict web content by country/region for compliance"，选项B是唯一同时满足这两项关键安全要求的解决方案。
