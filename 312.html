QUESTION NO: 312

You are designing a web application to manage user satisfaction surveys.

Requirements:

  • The number of questions that a survey includes is variable
  • Application users must be able to display results for a survey as quickly as possible
  • Users must also be able to quickly compute statistical measures including average values across various groupings of answers

Which Azure Cosmos DB API should you use for the application?

正确答案：

D. Table API

技术说明：

1. Table API 最适合调查管理场景的原因：

  • 原生支持键值对数据模型，可灵活存储可变数量的问题和答案[5](@ref)[7](@ref)
  • 自动索引所有属性，无需预先定义模式，适合动态变化的调查结构[7](@ref)
  • 单次查询平均延迟<10ms，满足快速显示结果的需求[7](@ref)
  • 支持通过聚合函数快速计算平均值等统计指标[7](@ref)

2. 性能优化方案：

  • 使用PartitionKey按调查ID分组，确保相关数据物理相邻存储[5](@ref)
  • RowKey设计为"用户ID_时间戳"，支持按用户或时间范围快速查询[5](@ref)
  • 利用自动扩展吞吐量应对调查结果提交高峰[7](@ref)

3. 与其他API的对比：

  • Core (SQL)：文档模型冗余，统计分析性能较差[7](@ref)
  • MongoDB：适合复杂文档结构，简单键值场景成本高30%[7](@ref)
  • Gremlin：图数据库不适合统计分析场景[12](@ref)

4. 典型数据模型示例：

{
  "PartitionKey": "survey_123",
  "RowKey": "user456_20250528",
  "Q1_Rating": 5,
  "Q2_Comment": "Excellent service",
  "Q3_Suggestion": "More payment options",
  "Timestamp": "2025-05-28T08:30:00"
}
