QUESTION NO: 13
You need to implement the bindings for the CheckUserContent function.
How should you complete the code segment?

需求背景

  • 用户上传内容到Blob存储后需要触发内容检查
  • 必须创建名为CheckUserContent的Azure函数
  • 函数需要访问上传的内容进行分析
  • 分析结果需要持久化存储
  • 必须最小化Azure服务成本
  • 所有服务调用必须使用Azure AD认证
01public static class CheckUserContent
02{
03 [FunctionName("CheckUserContent")]
04 public static void Run(
05 string content,
06 Stream output)
07 {
08 // 内容分析逻辑
09 }

答案与解析
正确答案:
绑定1: [BlobTrigger("userContent/{name}")] string content
绑定2: [Blob("userContent/{name}", FileAccess.Write)] Stream output

详细解析:

完整代码实现：

public static class CheckUserContent
{
    [FunctionName("CheckUserContent")]
    public static void Run(
        [BlobTrigger("userContent/{name}")] string content,
        [Blob("userContent/{name}", FileAccess.Write)] Stream output,
        ILogger log)
    {
        // 1. 分析内容
        var analysisResult = ContentAnalysisService.Analyze(content);
        
        // 2. 如果内容不合适，写入拒绝消息
        if (analysisResult.NeedsReview)
        {
            using (var writer = new StreamWriter(output))
            {
                writer.Write("此内容已被标记为需要审核");
            }
        }
    }
}

绑定配置原理：

1. BlobTrigger绑定：

  • 当用户上传新内容到"userContent"容器时自动触发函数
  • 使用{name}通配符捕获文件名
  • 绑定到string content参数获取文件内容
  • 符合"上传后立即触发检查"的需求
  • 成本最低的触发方式（无额外服务）

2. Blob输出绑定：

  • 允许函数修改原始Blob内容
  • 当内容需要审核时，替换为拒绝消息
  • 使用Stream参数提供高效写入能力
  • 满足"分析结果需要持久化存储"的要求

为什么不是其他选项？

QueueTrigger：

  • 需要额外步骤将上传事件加入队列
  • 增加系统复杂性和成本（需要队列服务）
  • BlobTrigger更直接高效

CosmosDBTrigger：

  • 适用于文档数据库变更
  • 但用户内容存储在Blob中，不是CosmosDB
  • 增加不必要的数据库成本

Table绑定：

  • 适合结构化数据存储
  • 但无法直接修改原始Blob内容
  • 不符合"替换内容"的需求
结论：BlobTrigger + Blob输出绑定是最佳组合，直接响应上传事件并修改内容，满足所有业务需求且成本最低。
