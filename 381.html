Azure Cosmos DB Always Encrypted 配置测试

QUESTION NO: 381

您正在开发一个将部署在 Azure 中的 Java 应用程序。该应用程序在 Azure Cosmos DB 中存储敏感数据。 您需要配置 Always Encrypted 以在应用程序内部加密敏感数据。首先应该做什么？
A. 创建客户管理的密钥 (CMK) 并将其存储在新的 Azure Key Vault 实例中
B. 创建 Azure AD 托管标识并将其分配给新的 Azure Key Vault 实例
C. 使用 Azure Cosmos DB SDK 创建数据加密密钥 (DEK) 并将密钥存储在 Azure Cosmos DB 中
D. 创建一个新容器，其中包含一个加密策略，指定要加密的 JSON 属性
正确答案: A
说明:

在 Azure Cosmos DB 中实现 Always Encrypted（客户端加密）需要以下步骤：

  1. 创建客户管理的密钥 (CMK) 并存储在 Azure Key Vault 中（这是第一步）
  2. 配置应用程序在写入数据前使用该密钥进行加密
  3. 配置应用程序在读取数据时使用该密钥进行解密

选项 B 错误：托管标识用于安全访问 Key Vault，但应在创建密钥之后配置

选项 C 错误：数据加密密钥 (DEK) 通常由客户端生成，但不应存储在 Cosmos DB 中

选项 D 错误：容器级别的加密策略用于服务端加密，而 Always Encrypted 要求客户端加密

因此，首先需要创建客户管理的密钥并存储在 Key Vault 中，为加密操作提供基础。
