问题37 - Policy Service 缩放策略

问题描述

您需要确保解决方案能够满足 Policy Service 的缩放要求。

QUESTION NO: 37

You need to ensure that the solution can meet the scaling requirements for Policy Service.

Which Azure Application Insights data model should you use?

an Application Insights dependency
an Application Insights event
an Application Insights trace
an Application Insights metric

正确答案

正确答案是：D. an Application Insights metric

解释说明

根据Proseware案例文档中的要求：

  • Policy service要求："Must use Application Insights to automatically scale with the number of policy actions"（必须使用Application Insights根据策略操作数量自动缩放）
  • 自动缩放机制：Azure的自动缩放功能通常基于指标（metrics）进行决策
  • 指标特性：Application Insights metrics提供数值型数据（如请求率、CPU使用率等），最适合用于自动缩放决策
  • 其他选项分析：
    • 依赖项（dependency）：跟踪外部服务调用，不适合缩放决策
    • 事件（event）：记录离散事件，不提供连续数值数据
    • 跟踪（trace）：用于诊断日志，不适合缩放决策

因此，选择Application Insights metric作为缩放依据最符合Policy Service的要求，能够根据策略操作数量动态调整服务规模。
