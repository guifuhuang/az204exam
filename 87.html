QUESTION NO: 87 HOTSPOT

You are developing an ASP.NET Core web application. You plan to deploy the application to Azure Web App for Containers.

The application needs to store runtime diagnostic data that must be persisted across application restarts. You have the following code:

public void SaveDiagData(string data) { var path = Environment.GetEnvironmentVariable("DIAGDATA") File.WriteAllText(Path.Combine(path, "data"), data); }

You need to configure the application settings so that diagnostic data is stored as required.

How should you configure the web app's settings? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

App setting Value
true
DIAGDATA

正确答案:

  • WEBSITES_ENABLE_APP_SERVICE_STORAGE: true
  • DIAGDATA: D:\home

说明:

要在Azure Web App for Containers中持久化存储诊断数据，需要正确配置以下设置：

  1. WEBSITES_ENABLE_APP_SERVICE_STORAGE必须设置为true，这样才能启用持久化存储功能。
  2. DIAGDATA应该设置为D:\home，因为这是Azure Web App中持久化存储的标准路径。

其他设置说明：

  • LOCALAPPDATA: 与持久化存储无关
  • WEBSITE_LOCALCACHE_ENABLED: 用于本地缓存，不是持久化存储
  • DOTNET_HOSTING_OPTIMIZATION_CACHE: 用于优化缓存，不是持久化存储
  • D:\local是临时存储路径，重启后会丢失数据

参考文档: Azure Web App持久化存储文档
