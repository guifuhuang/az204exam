You are developing an Azure solution to collect point-of-sale (POS) device data from 2,000 stores located throughout the world. A single device can produce 2 megabytes (MB) of data every 24 hours. Each store location has one to five devices that send data.

You must store the device data in Azure Blob storage. Device data must be correlated based on a device identifier. Additional stores are expected to open in the future.

You need to implement a solution to receive the device data.

Proposed Solution: Provision an Azure Event Hub. Configure the machine identifier as the partition key and enable capture.

Does the solution meet the goal?

正确答案: A. Yes

说明: 这个解决方案完全符合题目要求：

  1. Azure Event Hub 能够处理全球分布的2000个商店（每个商店1-5个设备）产生的数据量（每天2MB/设备）[4](@ref)[9](@ref)
  2. 将设备标识符配置为分区键可以确保来自同一设备的数据被发送到同一分区，保持数据相关性[4](@ref)[6](@ref)
  3. 启用捕获功能可以将事件数据自动保存到Azure Blob存储，满足存储要求[5](@ref)
  4. Event Hub具有良好的扩展性，可以轻松应对未来新增商店的需求[4](@ref)[9](@ref)

Event Hub的分区键机制确保相同设备的数据始终进入同一分区，而捕获功能则实现了自动将数据持久化到Blob存储的需求。这种架构特别适合这种全球分布式设备数据收集场景[4](@ref)[5](@ref)[6](@ref)。
