安全配置

Azure Logic App 安全配置测试题

案例背景

您需要为运输应用程序的 Azure Logic App 配置安全设置。该Logic App处理海洋运输和内陆运输工作流，需要支持X12消息格式，并确保资源安全连接到公司VNet。

QUESTION NO: 3
您需要保护 Shipping Logic App。
应该使用什么？
安全要求： 需要将资源安全连接到公司VNet，使用专用存储资源，并维护本地连接以支持遗留应用程序。
A
Azure App Service Environment (ASE)

提供隔离环境运行App Service应用，但并非专门为Logic App设计

B
Azure AD B2B integration

用于企业间协作的身份验证解决方案，不提供网络隔离

C
Integration Service Environment (ISE)

为Logic App提供专用环境，可直接部署到VNet中

D
VNet service endpoint

用于将Azure服务安全连接到VNet，但不提供专用环境

正确答案与解析

正确答案：C. Integration Service Environment (ISE)

解析说明：

根据文档中的要求，Shipping Logic App需要满足以下安全要求：

  • 将资源安全连接到公司VNet
  • 使用专用存储资源
  • 维护本地连接以支持遗留应用程序

Integration Service Environment (ISE) 是专门为Logic App设计的解决方案：

  • 提供专用环境运行Logic App，与多租户环境隔离
  • 可以直接部署到Azure虚拟网络(VNet)中
  • 支持固定成本模型，满足专用资源需求
  • 提供与本地系统的连接能力，支持混合环境
  • 满足X12消息格式支持要求

其他选项分析：

  • A. Azure App Service Environment (ASE)：主要用于Web应用，不是Logic App的最佳选择
  • B. Azure AD B2B integration：解决身份验证问题，但不提供网络隔离或VNet集成
  • D. VNet service endpoint：提供网络级安全，但不提供专用环境或本地连接支持

因此，ISE是满足所有要求的最佳解决方案。
