Azure API Management 配置测试题

问题: 您正在开发多个托管在Azure API Management (APIM)上的API。

API有以下要求：

  • 所有API都需要订阅密钥才能访问。
  • 包含订阅者必须接受的条款和条件。
  • 管理员必须审核并接受或拒绝订阅尝试。
  • 限制多个同时订阅的数量。

您需要实现这些API。应该怎么做？

A. 创建并发布一个产品。
B. 配置并应用基于查询字符串的版本控制。
C. 配置并应用基于标头的版本控制。
D. 为所有API添加一个新修订版。将修订版设为当前版本并添加更改日志条目。

答案及解析

正确答案: A. 创建并发布一个产品。

解析:

  • 创建并发布一个产品：产品是API的组合，可以设置订阅密钥、条款和条件、订阅审批等。这满足了所有API需要订阅密钥、包含条款和条件、管理员审核订阅尝试的要求。
  • 基于查询字符串的版本控制：这与版本控制相关，但不涉及订阅密钥、条款和条件、订阅审批等要求。
  • 基于标头的版本控制：这也与版本控制相关，但不涉及订阅密钥、条款和条件、订阅审批等要求。
  • 为新修订版添加更改日志条目：这与API版本控制和更改日志相关，但不涉及订阅密钥、条款和条件、订阅审批等要求。

因此，创建并发布一个产品是最符合要求的解决方案。
