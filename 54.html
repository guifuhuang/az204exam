Topic 7: 零售店位置Azure函数配置

案例背景

VanArsdel Ltd.是一家全球办公用品公司，总部位于加拿大，在全球设有零售店。公司正在开发多个基于云的解决方案，以支持其商店、分销商、供应商和配送服务。

零售店位置要求：

  • 当数据上传到Blob存储时，Azure Functions必须立即处理数据
  • Azure Functions必须使用本机SQL语言查询更新Azure Cosmos DB
  • 每晚审核商店销售交易信息以验证数据、处理销售财务和核对库存
  • 由于意外删除数据，必须执行零售店位置数据的时点恢复
  • Azure Cosmos DB查询显示出高请求单位(RU)使用率，并且包含多个复杂查询，在函数应用扩展时对大项表现出高读取延迟

问题 54: 零售店位置Azure函数配置

您需要实现零售店位置的Azure Function。应该如何配置解决方案？在下方选择适当的选项。

配置项 值
Binding
Binding Direction
Trigger

正确答案及说明

✓ Binding: Blob storage
✓ Binding Direction: Input
✓ Trigger: Blob storage

说明：

根据案例要求，Azure Functions需要在数据上传到Blob存储时立即触发处理，因此Trigger选择Blob storage。

函数需要读取Blob存储中的数据，因此需要配置一个输入绑定（Input Binding）到Blob storage，所以Binding选择Blob storage，Binding Direction选择Input。

这种配置满足以下要求：

  • 当数据上传到Blob存储时立即处理（通过Blob storage触发器实现）
  • 读取Blob存储中的数据（通过Blob storage输入绑定实现）
  • 实现高效的数据处理流程，为后续的Cosmos DB更新提供基础

文档依据：

Azure Functions文档明确指出：

  • Blob存储触发器用于在blob添加或更新时启动函数
  • Blob输入绑定用于在函数中读取blob存储数据
  • 对于更新Cosmos DB的需求，应使用Cosmos DB输出绑定（但本题配置的是输入处理部分）
