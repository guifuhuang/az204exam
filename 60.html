Topic 7: 配送服务遥测数据配置

配送服务要求

VanArsdel Ltd.需要实现配送服务的遥测数据存储解决方案。

具体要求：

  • 使用Azure Function将服务遥测数据存储在Azure Cosmos DB中
  • 数据必须包含：物品ID、配送车辆牌照、车辆包裹容量、当前车辆位置坐标
  • 存储配送司机配置文件信息到Azure Active Directory
  • 使用Azure AD进行身份验证和授权
  • 尽可能使用托管身份

遥测数据结构：

字段名称 数据类型 描述
Item ID 字符串 物品的唯一标识符
Vehicle license plate 字符串 配送车辆牌照号码
Vehicle package capacity 整数 车辆最大包裹容量
Vehicle location coordinates 地理坐标 车辆当前位置的经纬度

问题 60: 配送服务遥测数据配置

您需要实现配送服务遥测数据。应如何配置解决方案？

注意：每个正确选项值1分。

Azure Cosmos DB 配置项 值
API
Partition Key

您的选择：

API: 未选择

分区键: 未选择

正确答案及说明

参考答案正确：API: Core (SQL), 分区键: Vehicle license plate

✓ API: Core (SQL)
✓ 分区键: Vehicle license plate

说明：

根据案例要求和Azure最佳实践，以下是配置选择的理由：

API选择：Core (SQL)

  • Core (SQL) API是Azure Cosmos DB的默认API，提供最全面的功能和最佳性能
  • 支持SQL查询语法，适合结构化数据查询
  • 提供丰富的索引功能，可以优化查询性能
  • 与其他API相比，Core (SQL) API在功能和生态系统支持方面最为成熟

分区键选择：Vehicle license plate

  • 车辆牌照具有较高的基数（不同值的数量），有助于数据均匀分布
  • 大多数查询可能基于特定车辆，因此按车辆分区可以提高查询效率
  • 车辆位置坐标变化过于频繁，不适合作为分区键
  • 车辆包裹容量值范围有限，可能导致分区热点问题
  • 物品ID虽然唯一，但可能导致分区过多，且与查询模式不匹配

数据模型设计建议：

{
  "id": "unique-id",
  "itemId": "item-12345",
  "vehicle": {
    "licensePlate": "ABC123",
    "packageCapacity": 200,
    "location": {
      "type": "Point",
      "coordinates": [47.6062, -122.3321]
    }
  },
  "timestamp": "2023-08-15T12:34:56Z"
}

为什么这个配置是最佳的？

  • 满足案例中"使用Azure Function存储数据"的要求
  • Core (SQL) API支持地理空间查询，适合位置坐标数据
  • 车辆牌照作为分区键可以确保数据均匀分布
  • 此配置提供良好的可扩展性和查询性能
