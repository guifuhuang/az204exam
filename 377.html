Azure App Configuration 多选题

QUESTION NO: 377
您开发了一个使用Azure App Configuration的ASP.NET Core应用程序。您还创建了一个包含100个设置的App Configuration。该应用程序必须满足以下要求：

要求：

  • 确保在单个设置发生更改时所有配置数据的一致性
  • 动态处理配置数据更改而不导致应用程序重启
  • 减少对App Configuration API的总体请求数
您必须在应用程序中实现动态配置更新。以下哪两种方式可以实现此目标？（每个正确答案代表解决方案的一部分）
答案与解析
正确答案：A 和 E

解析：

A. 增加App Configuration缓存过期时间：通过增加缓存过期时间，可以减少应用程序向App Configuration API发出的请求次数。较长的缓存时间意味着应用程序不会频繁检查配置更新，从而减少API调用次数，满足"减少总体请求数"的要求。

E. 使用标记键(sentinel key)：

  • 创建一个特殊的标记键（如"version"或"last_updated"）来监控配置变化
  • 当标记键的值发生变化时，触发所有配置的刷新（refreshAll=true）
  • 这种方法确保所有配置同时更新，保持数据一致性
  • 应用程序只需监控一个键的变化，而不是所有100个键，大大减少请求数量
  • 配置更改可以动态处理，无需重启应用

为什么不选择其他选项？

B. 环境变量：环境变量是静态的，无法实现动态更新，且不适用于大量配置

C. 减少缓存时间：会增加API请求次数，与"减少请求数"的要求相悖

D. 注册所有键但refreshAll=false：无法确保所有配置同时更新，可能导致数据不一致

F. Key Vault：用于存储机密，与App Configuration动态更新机制无关
