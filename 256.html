Scenario:

You have an Azure App Service web app named App1 that uses a deployment slot named Development. You create additional deployment slots named Testing and Production. You enable auto swap on the Production deployment slot.

Goal:

You need to ensure that scripts run and resources are available before a swap operation occurs.

Solution:

Update the app with a method named statuscheck to run the scripts. Update the app settings for the app. Set the WEBSITE_SWAP_WARMUP_PING_PATH and WEBSITE_SWAP_WARMUP_PING_STATUSES with a path to the new method and appropriate response codes.

Question:

Does the solution meet the goal?

正确答案: A. Yes

说明:

这个解决方案是正确的。在Azure应用服务中，WEBSITE_SWAP_WARMUP_PING_PATH和WEBSITE_SWAP_WARMUP_PING_STATUSES是两个重要的应用设置，用于控制交换操作前的预热行为。

WEBSITE_SWAP_WARMUP_PING_PATH指定了交换操作前要调用的路径（这里是statuscheck方法），而WEBSITE_SWAP_WARMUP_PING_STATUSES定义了可接受的HTTP响应代码。通过这种方式，可以确保在交换操作发生前，脚本能够运行并且资源可用。

这种方法正是Azure推荐的在交换操作前验证应用是否准备就绪的方式，因此该解决方案能够满足题目要求的目标。
