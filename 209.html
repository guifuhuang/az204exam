QUESTION NO: 209

Scenario:

You are developing a medical records document management website. The website is used to store scanned copies of patient intake forms. If the stored intake forms are downloaded from storage by a third party, the content of the forms must not be compromised.

Requirement:

You need to store the intake forms according to the requirements.

Solution:

Store the intake forms as Azure Key Vault secrets.

Question:

Does the solution meet the goal?

Answer:

B. No

解释：

这个解决方案不能满足需求，因为Azure Key Vault不是设计用来存储医疗记录文档的。

主要原因：

  • Azure Key Vault主要用于存储小型敏感数据，如API密钥、密码、连接字符串等，而不是大型文档或扫描件[1](@ref)[2](@ref)[3](@ref)
  • Key Vault有存储大小限制，单个secret最大只能存储25KB，不适合存储扫描的医疗表格[10](@ref)
  • Key Vault的设计目的是为服务器应用程序存储配置机密，而不是用户数据[9](@ref)
  • 医疗记录文档应该存储在专门的医疗数据存储解决方案中，如加密的Azure Blob存储[9](@ref)

正确的解决方案应该是：

  1. 使用Azure Blob存储来存储扫描的医疗表格
  2. 启用Azure存储服务加密(SSE)来保护静态数据
  3. 使用客户管理的密钥(CMK)进行额外保护
  4. 实施严格的访问控制和审计日志

补充说明：

虽然Key Vault不适合直接存储医疗文档，但它可以用来安全地存储访问这些文档所需的加密密钥或连接字符串[8](@ref)[9](@ref)。
