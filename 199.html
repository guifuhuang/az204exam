QUESTION NO: 199

You are preparing to deploy an ASP.NET Core website to an Azure Web App from a GitHub repository. The website includes static content generated by a script.

You plan to use the Azure Web App continuous deployment feature.

You need to run the static generation script before the website starts serving traffic.

What are two possible ways to achieve this goal? Each correct answer presents a complete solution.

NOTE: Each correct selection is worth one point.

正确答案:

A. Create a file named .deployment in the root of the repository that calls a script which generates the static content and deploys the website.

B. Add a PreBuild target in the websites csproj project file that runs the static content generation script.

说明:

在Azure Web App持续部署中，有两种主要方法可以在网站开始处理流量前运行静态内容生成脚本：

  1. 选项A (.deployment文件):
    • 在仓库根目录创建.deployment文件是Azure Web App自定义部署的标准方法
    • 可以指定部署前运行的脚本（[config]部分设置SCM_PRE_BUILD_COMMAND或PRE_BUILD_COMMAND）
    • 示例.deployment文件内容：
      [config] command = ./generate-static-content.sh && ./deploy.sh
    • 确保脚本在部署过程中执行，生成静态内容
  2. 选项B (PreBuild目标):
    • 在.csproj文件中添加PreBuild目标会在构建过程中自动执行
    • 这是ASP.NET Core项目的标准构建扩展点
    • 示例.csproj配置：
      <Target Name="PreBuild" BeforeTargets="PreBuildEvent"> <Exec Command="generate-static-content.cmd" /> </Target>
    • 确保静态内容在发布前生成并包含在部署包中

其他选项不正确的原因:

  • 选项C (run.cmd): /run/run.cmd不是Azure Web App识别的标准部署钩子文件
  • 选项D (WEBSITE_RUN_FROM_PACKAGE): 这个设置用于从ZIP包运行应用，与运行构建脚本无关[8](@ref)

关键点:

  • Azure Web App持续部署支持通过.deployment文件自定义部署步骤
  • ASP.NET Core项目可以通过.csproj文件中的构建目标扩展构建过程
  • 静态内容生成应该在部署过程中或构建过程中完成
  • WEBSITE_RUN_FROM_PACKAGE设置适用于函数应用，不适用于此场景[8](@ref)

关于参考答案A,D的说明:

原参考答案A,D不完全正确。选项D提到的WEBSITE_RUN_FROM_PACKAGE设置主要用于Azure Functions应用从部署包运行[8](@ref)，而不是用于在Web App中运行构建脚本。正确的两个选项应该是A和B。
