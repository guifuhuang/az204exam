Azure Service Bus 安全配置测试题

Topic 8: Munson's Pickles and Preserves Farm

问题 66: 解决开发人员的身份验证错误

场景描述

Munson's Pickles and Preserves Farm 正在将应用程序从本地数据中心迁移到 Microsoft Azure。开发人员在本地调试应用程序时遇到 Service Bus 的身份验证错误。

相关要求

  • 所有站点设置必须集中存储、安全且不使用密钥
  • 必须使用 Azure 资源的托管标识来支持 Microsoft Entra ID 身份验证的服务
  • 在提供任何用户权限或进程访问权限时，必须应用最小权限原则

具体问题

开发人员在本地调试时收到 Service Bus 的身份验证错误。

Service Bus 安全配置

正确答案与解析

正确配置:

RBAC 角色: Service Bus Data Owner

作用域: Queue

解析:

在本地调试环境中，开发人员需要适当的权限来访问 Service Bus：

  • Service Bus Data Owner 角色 提供对 Service Bus 资源的完全访问权限（包括发送和接收消息），这解决了开发人员在调试期间可能需要的各种操作。
  • 将作用域设置为 Queue 遵循最小权限原则，仅授予开发人员访问特定队列的权限，而不是整个命名空间或资源组。
  • 此配置允许开发人员在本地使用 Azure AD 凭据进行身份验证，同时满足公司安全要求（不使用密钥、应用最小权限原则）。
  • 使用托管标识在生产环境中运行的服务上进行身份验证，但开发人员本地调试时需要使用 Azure AD 用户账户进行身份验证。

其他选项分析：

  • Owner/Contributor：权限过大，包含管理操作权限，不符合最小权限原则
  • Service Bus Data Sender：仅提供发送权限，不满足调试时可能需要的接收/管理需求
  • Namespace/Resource group：作用域过大，不符合最小权限原则
