QUESTION NO: 223

Scenario:

You develop and deploy an ASP.NET web app to Azure App Service. You use Application Insights telemetry to monitor the app.

Requirements:

  1. Test the app to ensure it is available and responsive from various points around the world
  2. Perform tests at regular intervals
  3. Send alerts to support staff if the app is not responding

Task:

Which two test types can you use? Each correct answer presents a complete solution.

NOTE: Each correct selection is worth one point.

Answer:

正确答案是：B. multi-step web 和 C. URL ping

Explanation:

在Application Insights中，有以下四种可用性测试类型：

  1. 标准测试：通过发送单个请求来检查网站可用性
  2. 自定义TrackAvailability测试：创建自定义应用程序运行可用性测试
  3. URL ping测试：验证端点是否响应并测量性能
  4. 多步骤Web测试：模拟用户会话并验证多个步骤的响应

选择理由：

  • URL ping测试(C)：
    • 最简单的可用性测试类型
    • 可以从全球多个位置定期发送HTTP请求
    • 可以配置响应时间阈值和内容匹配规则
    • 支持自动警报功能
  • 多步骤Web测试(B)：
    • 模拟更复杂的用户交互流程
    • 可以记录和回放多个HTTP请求序列
    • 支持验证每个步骤的响应
    • 同样支持从全球多个位置执行和警报功能

其他选项分析：

  • A. integration：集成测试不是Application Insights提供的可用性测试类型
  • D. unit：单元测试是开发阶段的测试方法，不适用于生产环境监控
  • E. load：负载测试用于评估系统在压力下的性能，不是针对可用性监控

配置建议：

  1. 对于简单监控，使用URL ping测试配置：
    • 设置5个以上的测试位置
    • 测试频率建议5分钟
    • 配置响应超时阈值(如30秒)
    • 设置内容匹配规则验证关键页面内容
  2. 对于关键业务流程，使用多步骤Web测试：
    • 记录用户登录-操作-登出的完整流程
    • 验证每个步骤的关键响应
    • 配置更严格的警报规则(如3/5位置失败)

警报配置：

  • 在创建测试后会自动启用基本警报
  • 建议配置操作组发送邮件/SMS/Teams通知
  • 对于关键应用，建议设置"Y个位置中有X个报告失败"的警报规则
