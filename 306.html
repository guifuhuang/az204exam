QUESTION NO: 306

You are developing an Azure Function App that generates end of day reports for retail stores. All stores close at 11 PM each day. Reports must be run one hour after closing. You configure the function to use a Timer trigger that runs at midnight. Customers in the Western United States Pacific Time zone (UTC - 8) report that the Azure Function runs before the stores close.

You need to ensure that the Azure Function runs at midnight in the Pacific Time zone.

What should you do?

正确答案： B. Add an app setting named WEBSITE_TIME_ZONE that uses the value Pacific Standard Time

说明：

这个问题涉及到Azure Function的时区设置。题目描述的情况是，定时触发器在UTC时间午夜运行，但对于太平洋时区(UTC-8)的用户来说，这相当于下午4点（商店还没关门）。

解决方案是通过添加名为WEBSITE_TIME_ZONE的应用设置，并将其值设为"Pacific Standard Time"，这样定时触发器就会按照太平洋时区的时间来执行，而不是默认的UTC时间。

其他选项分析：

  • A选项：更改Azure Function的运行区域不会影响定时触发器的时区设置
  • C选项：虽然改为7AM UTC确实会在太平洋时区的午夜运行，但这不是一个灵活的解决方案
  • D选项：升级到Premium计划与时区问题无关

因此，B选项是最佳解决方案，它允许Function按照太平洋时区的午夜时间执行。
