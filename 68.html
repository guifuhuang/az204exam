Azure App Service 配置测试题

Topic 8: Munson's Pickles and Preserves Farm

问题 68: 配置App Service以支持企业网站迁移

场景描述

Munson's Pickles and Preserves Farm 正在将企业网站迁移到 Azure App Service。迁移需要满足特定的业务和安全要求，包括成本优化、自动扩展和代码验证流程。

相关要求

  • 最小化 Azure 托管成本
  • 应用程序必须独立于计算资源自动扩展
  • 所有代码更改必须由内部人员在发布到生产环境之前进行验证
  • 在验证期间提供直接链接回生产应用程序
  • 集中存储和保护所有站点设置，不使用密钥

当前挑战

农民报告在高CPU/内存使用期间出现HTTP 503错误，分销商报告在高响应时间和网络流量期间出现HTTP 502错误。

App Service 配置选项

正确答案与解析

正确配置:

App Service 计划: Standard

代码更改验证功能: 部署槽位

解析:

此配置满足所有迁移要求：

  • Standard 计划 提供自动扩展能力（解决HTTP 503/502错误），同时保持成本优化（满足"Minimize Azure hosting costs"要求）
  • 部署槽位 允许内部人员在发布到生产环境前验证代码更改（满足"Validate app/API updates before production release"要求）
  • 部署槽位提供直接链接到暂存环境的功能（满足"Provide direct link back to production app during validation"要求）
  • App Service支持集中存储站点设置（应用设置）并通过托管标识保护访问（满足"Centrally store and secure all site settings without using secrets"要求）

其他选项分析：

  • Basic计划：不支持自动扩展，无法解决高负载问题
  • Premium/Isolated计划：成本过高，不符合成本优化要求
  • 自定义容器：与代码验证流程无关
  • 域名证书：用于TLS配置，不解决验证需求
  • 部署凭据：用于FTP部署，不提供验证环境
