QUESTION NO: 281 DRAG DROP

You develop an application. You plan to host the application on a set of virtual machines (VMs) in Azure.

You need to configure Azure Monitor to collect logs from the application.

Which four actions should you perform in sequence? To answer, move the appropriate actions from the list of actions to the answer area and arrange them in the correct order.

Actions
Create a Log Analytics workspace.
Install agents on the VM and VM scale set to be monitored.
Send console logs.
Add a VM insights solution.
Create an Application Insights resource.
Answer Area
1
2
3
4

正确答案:

1
Create a Log Analytics workspace.
2
Add a VM insights solution.
3
Install agents on the VM and VM scale set to be monitored.
4
Create an Application Insights resource.

说明:

配置Azure Monitor收集应用程序日志的正确顺序如下：

  1. 创建Log Analytics工作区：首先需要创建工作区作为日志数据的存储和分析中心。
  2. 添加VM见解解决方案：在Log Analytics工作区中安装VMInsights解决方案，这是使用VM见解的必要条件。
  3. 在虚拟机上安装代理：安装Log Analytics代理和依赖代理以收集数据，必须在工作区配置完成后进行。
  4. 创建Application Insights资源：用于收集应用程序级别的日志和性能数据，这是配置的最后一步。

这个顺序确保了从基础设施准备到应用程序监控的完整流程：

  • 首先建立日志存储基础(Log Analytics工作区)
  • 然后添加虚拟机监控解决方案(VM Insights)
  • 接着在虚拟机上部署监控代理
  • 最后配置应用程序级别的监控(Application Insights)

注意：发送控制台日志(Send console logs)不是配置过程中的独立步骤，而是通过上述配置自动实现的。
