QUESTION NO: 253 HOTSPOT

You are developing an application that uses a premium block blob storage account. You are optimizing costs by automating Azure Blob Storage access tiers.

You apply the following policy rules to the storage account. You must determine the implications of applying the rules to the data. (Line numbers are included for reference only.)

01 {
02 "rules": [
03 {
04 "name": "agingDataRule",
05 "enabled": true,
06 "type": "Lifecycle",
# Question Yes No
1 Block blobs prefixed with container1/salesorders or container2/inventory which have not been modified over 60 days are moved to cool storage.Blobs that have not been modified over 120 days are archived to the archive tier.
2 Blobs are moved to cool storage if they have not been accessed for 30 days.
3 Blobs are automatically tiered from cool back to hot if accessed again after being tiered to cool.
4 All block blobs older than 730 days will be deleted.

Correct Answer and Explanation

Correct Answers:

  1. Yes - Lifecycle management can move blobs to cooler tiers based on modification time
  2. Yes - Lifecycle management can move blobs based on last access time
  3. Yes - Azure automatically moves blobs back to hot tier when accessed
  4. No - The policy snippet doesn't show any deletion rules

中文说明：

1. 带有container1/salesorders或container2/inventory前缀的块blob在60天未修改后会移动到冷存储？120天未修改的blob会被归档？

  • 答案：Yes
  • 原因：生命周期管理可以根据修改时间将blob移动到更冷的层级
  • 参考文档：Microsoft生命周期管理文档

2. 如果blob 30天未被访问会被移动到冷存储？

  • 答案：Yes
  • 原因：生命周期管理可以根据最后访问时间移动blob
  • 需要在存储账户上启用"访问跟踪"功能

3. blob从冷存储被访问后会自动回到热存储？

  • 答案：Yes
  • 原因：当访问冷存储或存档存储中的blob时，Azure会自动将其移回热存储
  • 这是Azure的默认行为，不需要特别配置

4. 所有超过730天的块blob会被删除？

  • 答案：No
  • 原因：提供的策略片段中没有显示任何删除规则
  • 需要明确的delete动作定义才能实现自动删除

技术说明：

  • 生命周期管理规则可以基于修改时间或访问时间
  • 自动回热是Azure的默认行为，无需特别配置
  • 存档层转换需要明确的tierToArchive动作定义
  • 删除操作需要明确的delete动作定义
  • 基于前缀的规则需要定义prefixMatch

完整生命周期策略示例：

{
  "rules": [
    {
      "name": "agingRule",
      "enabled": true,
      "type": "Lifecycle",
      "definition": {
        "actions": {
          "baseBlob": {
            "tierToCool": { "daysAfterModificationGreaterThan": 60 },
            "tierToArchive": { "daysAfterModificationGreaterThan": 120 },
            "delete": { "daysAfterModificationGreaterThan": 730 }
          }
        },
        "filters": {
          "blobTypes": ["blockBlob"],
          "prefixMatch": ["container1/salesorders", "container2/inventory"]
        }
      }
    }
  ]
}
