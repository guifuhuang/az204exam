QUESTION NO: 74
HOTSPOT

You need to correct the errors for farmers and distributors.

Which solution should you use? To answer, select the appropriate options in the answer area.

NOTE: Each correct selection is worth one point.

Farmers' Errors

Farmers report HTTP 503 errors during high CPU/memory usage.

这些错误发生在高CPU/内存使用期间，表示服务不可用。

Distributors' Errors

Distributors report HTTP 502 errors during high response times and network traffic.

这些错误发生在高响应时间和网络流量期间，表示网关错误。

正确答案与解析

正确解决方案:

Farmers' Errors:

升级到高级App Service计划 (Scale up the App Service plan to Premium)

Distributors' Errors:

创建自定义自动缩放规则以增加实例数量 (Create a custom autoscale rule to increase the instance count)

问题背景分析

农民的问题 (HTTP 503):

  • 发生在高CPU/内存使用期间
  • 表示服务器资源不足，无法处理请求
  • 根本原因是计算资源不足

分销商的问题 (HTTP 502):

  • 发生在高响应时间和网络流量期间
  • 表示网关或代理服务器从上游服务器收到无效响应
  • 根本原因是后端实例无法处理高并发请求

解决方案解析

农民问题的解决方案: 升级到高级App Service计划

  • HTTP 503错误直接与资源不足相关
  • 升级到高级计划提供更强的CPU和内存资源
  • 符合文档中"解决高CPU/内存使用问题"的要求
  • 提供专用资源，确保服务在高负载下可用

分销商问题的解决方案: 创建自定义自动缩放规则

  • HTTP 502错误与实例过载相关
  • 自动缩放规则可根据流量动态增加实例数量
  • 完全符合文档中"应用程序必须自动独立于计算资源扩展"的要求
  • 在高流量期间自动增加实例，处理更多请求
  • 在流量减少时自动缩减实例，优化成本

为什么其他选项不正确？

农民问题其他选项分析:

  • 添加部署槽：用于蓝绿部署，不解决资源不足问题
  • 配置本地缓存：优化磁盘I/O，但不解决CPU/内存瓶颈
  • 创建CDN：加速静态内容，但对动态API请求无效

分销商问题其他选项分析:

  • 升级到高级计划：纵向扩展，但无法处理流量突发（已用于农民问题）
  • 配置本地缓存：不解决实例不足问题
  • 重启应用程序：临时措施，不解决根本问题

实施建议

  • 为农民API实施高级App Service计划（P2v2或更高）
  • 为分销商API配置基于CPU使用率或请求队列的自动缩放规则
  • 设置最小2个、最大10个实例的缩放范围
  • 监控错误率并调整缩放阈值
