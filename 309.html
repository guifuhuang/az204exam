QUESTION NO: 309

You are developing an application to store information about the organizational structure for a company.

Users must be able to determine:

  • Which people report to a particular manager
  • The office where employees work
  • The projects that are assigned to an employee

Which Azure Cosmos DB API should you use for the application?

正确答案：

E. MongoDB

技术说明：

1. MongoDB API 最适合组织架构管理的原因：

  • 文档模型匹配数据结构：员工信息、汇报关系、办公地点和项目分配可以自然地表示为嵌套文档或引用
  • 灵活查询：支持丰富的查询语法（如$lookup聚合），可直接查询嵌套字段（如"查找某经理的所有下属"）
  • 开发效率高：与大多数现代开发框架（如Node.js、.NET）集成良好，减少代码复杂度
  • 性能优化：对层级数据的查询性能优于关系型模型（如Core/SQL）

2. 典型数据模型示例：

{
  "_id": "emp101",
  "name": "张三",
  "position": "经理",
  "reportsTo": "emp100",  // 汇报关系
  "office": {             // 办公地点
    "building": "A栋",
    "floor": 5,
    "desk": "502A"
  },
  "projects": [           // 分配的项目
    {"id": "proj1", "role": "负责人"},
    {"id": "proj2", "role": "成员"}
  ]
}

3. 其他API的限制：

  • Core (SQL)：需要复杂JOIN操作查询多级关系，性能较差
  • Cassandra：适合宽列存储，无法高效处理嵌套文档
  • Table API：仅支持简单键值对，无法表示复杂关系
  • Gremlin：虽然能处理关系，但对于简单层级结构过度设计

4. 解决方案优势：

  • 快速开发：文档模型与业务数据结构高度一致
  • 查询灵活性：支持从简单（如"查找某办公室员工"）到复杂（如"查找参与某项目的所有经理"）的查询
  • 无缝扩展：未来可轻松添加新字段（如员工技能、部门预算等）
  • 成本可控：支持按需扩展吞吐量，适合SaaS多版本需求
