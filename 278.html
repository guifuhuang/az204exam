QUESTION NO: 278

You are developing a solution that will use a multi-partitioned Azure Cosmos DB database. You plan to use the latest Azure Cosmos DB SDK for development.

The solution must meet the following requirements:

  • Send insert and update operations to an Azure Blob storage account.
  • Process changes to all partitions immediately.
  • Allow parallelization of change processing.

You need to process the Azure Cosmos DB operations.

What are two possible ways to achieve this goal? Each correct answer presents a complete solution.

NOTE: Each correct selection is worth one point.

正确答案：C 和 D

说明：

1. 选项C：使用Azure Functions的Cosmos DB触发器是实现变更订阅处理的推荐方式。触发器会自动处理多分区场景，并能并行处理变更。当配置为监听容器级别的变更时，它可以捕获所有分区的变更。

2. 选项D：使用FeedIterator和FeedRange对象可以实现手动拉取变更订阅，并通过FeedRange对象实现跨多个函数的并行处理。这种方法提供了更精细的控制，适合需要自定义处理逻辑的场景。

3. 选项A和B不正确的原因：

  • 选项A中的变更订阅估计器(change feed estimator)仅用于监控变更订阅的进度，不能直接用于处理变更。
  • 选项B虽然可以实现变更订阅处理，但不是最直接和高效的解决方案，相比Azure Functions需要更多的手动配置。
